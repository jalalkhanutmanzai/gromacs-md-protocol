# 04 — Energy Minimisation (EM)

Energy minimisation (EM) removes steric clashes and bad contacts introduced
during system building before the actual MD simulation starts.

---

## Why energy minimisation is necessary

After solvation and ion placement, some atoms may overlap or have unrealistic
bond lengths.  Running MD directly on such a structure would cause the
simulation to explode.  EM moves atoms to a local energy minimum using a
gradient descent algorithm (steepest descent) without simulating any dynamics.

---

## 4.1 MDP parameter file (`mdp/em.mdp`)

Key parameters for EM:

| Parameter | Value | Meaning |
|---|---|---|
| `integrator` | `steep` | Steepest-descent minimiser |
| `nsteps` | `50000` | Maximum number of steps |
| `emtol` | `1000.0` | Convergence: max force < 1000 kJ mol⁻¹ nm⁻¹ |
| `emstep` | `0.01` | Initial step size (nm) |
| `nstxout` | `500` | Write coordinates every 500 steps |
| `nstenergy` | `500` | Write energies every 500 steps |

---

## 4.2 Pre-process with `grompp`

`gmx grompp` (GROMACS pre-processor) combines the MDP, coordinate, and
topology files into a single binary run-input file (`.tpr`).

```bash
source config/config.env

mkdir -p work/03_em
cd work/03_em

gmx grompp \
    -f ../../mdp/em.mdp \
    -c ../02_solvate/ions.gro \
    -p ../01_topology/topol.top \
    -o em.tpr \
    -maxwarn 2
```

**Flags explained:**

| Flag | Meaning |
|---|---|
| `-f` | MDP parameter file |
| `-c` | Input coordinate file (from step 03) |
| `-p` | Topology file |
| `-o` | Output binary TPR file |
| `-maxwarn 2` | Allow up to 2 non-fatal warnings |

## 4.3 Run energy minimisation with `mdrun`

```bash
gmx mdrun \
    -v \
    -deffnm em \
    -gpu_id ${GPU_ID:-0} \
    -ntmpi 1 \
    -ntomp ${NTOMP:-4}
```

**Flags explained:**

| Flag | Meaning |
|---|---|
| `-v` | Verbose output (prints progress) |
| `-deffnm em` | Use `em` as prefix for all output files |
| `-gpu_id 0` | Use GPU 0 |
| `-ntmpi 1` | Use 1 MPI thread (thread_mpi build) |
| `-ntomp 4` | Use 4 OpenMP threads (adjust to your CPU core count) |

EM typically completes in **2–5 minutes** on a modern GPU.

---

## 4.4 Check convergence

The run has converged when you see output like:

```
Converged to machine precision in X steps,
but did not converge to requested Fmax < 1000.
or
Potential Energy  = -X.XXXE+06
Maximum force     = X.XXXE+02
```

A maximum force below 1000 kJ mol⁻¹ nm⁻¹ is the target.

Inspect the potential energy:

```bash
printf "Potential\n0\n" | gmx energy \
    -f em.edr \
    -o potential.xvg

# View with Grace or Python
# A steadily decreasing curve that levels off = convergence
```

## 4.6 Common problems

See [docs/09_TROUBLESHOOTING.md](09_TROUBLESHOOTING.md) for:

- `step X: Lincs warning` — LINCS constraint failure during EM
- `nan or inf` energies — atoms too close; increase emstep or check topology
- EM does not converge — increase `nsteps` or lower `emtol`
